The owner app is trying to select a barber with ID 'owner', but in Supabase, barbers have UUID IDs. The owner is always the first barber created during setup. This fix updates the code to properly select the first barber as the owner.

Problem

The owner app has this hardcoded line that causes the warning:

This tries to find a barber with ID 'owner', which doesn't exist in Supabase.

- window.currentBarberId = 'owner';
+ window.currentBarberId = null; // Will be set to first barber (owner)

function selectBarber(barberId) { // If no barberId provided, select the first barber (owner) if (!barberId && barbers.length > 0) { barberId = barbers[0].id; } window.currentBarberId = barberId; // Rest of the function stays the same... let barber = barbers.find(b => b.id === barberId); // If not found, use first barber instead of 'owner' if (!barber && barbers.length > 0) { console.warn(`Barber ${barberId} not found, using first barber (owner)`); barber = barbers[0]; window.currentBarberId = barber.id; } // Continue with rest of function... }

How to Apply the Fix

  1. Open owner-app.html in your code editor
  2. Search for window.currentBarberId = 'owner' (line 7127)
  3. Replace it with window.currentBarberId = null
  4. Find the selectBarber function (around line 7419)
  5. Add the null check at the beginning of the function
  6. Update the fallback logic to use barbers[0] instead of a hardcoded owner object
  7. Save the file

Testing

After applying the fix:

  • The "Barber owner not found" warning should disappear
  • The first barber (created during setup) will be selected automatically
  • The owner's name and services will display correctly
